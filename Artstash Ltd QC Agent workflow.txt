Reporting and process

Two styles of reporting depending on the frequency of QC agent usage. For internal purposes Artstash C will be sending assets to QC as and when work comes in

For external client usage, its likely vendors will complete all videos and then we run batches of creative through the QC agent in one go.

Both workflows need to be considered to ensure process is efficient and easy to use

End of week QC agent status report per client (Artstash Process + clients)
Total number of assets passed through QC agent this week
% of assets that passed QC
% of assets that failed QC
List of reasons for failing QC check and each of those need to link out to the asset that failed
End of batch upload report for batch QC process (agency and 3rd party usage)
Total number of assets passed through QC agent in batch upload
% of assets that passed QC
% of assets that failed QC
List of reasons for failing QC check and each of those need to link out to the asset that failed
